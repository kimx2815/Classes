1.1 Introduction

The intent of this lab is to make you more familiar with bit representations and bit operators in C.
The assignment involves solving a series of programming puzzles in which you will be restricted from using certain operations. They are artificial, (that is, you probably won’t be programming like this in the Real World,) but they will get you thinking deeply about bit representation and manipulation. All questions can be directed to the TAs or Prof. Zhai. Also, feel free to ask questions on the forum or contact us. Our goal is to maximize your understanding of the class materials.

1.2 Rules and Notes

  • You may work in pairs on this assignment. Groups of greater than two are not permitted.

  • Use the submit utility to turn in your code.

  • This lab includes the grading program that I will use. You should be able to know your approximate grade before you submit your program.

For this lab, you will submit the following file:

2.2 Your task

Carefully read the instructions in bits.c. The general idea is as follows: Code up the functions as defined, using only the operators allowed for each function.

Caveats

  • You may not use loops, conditionals, or other C functions.

  • You will be limited to a certain number of operators for each function.

  • You may not use constants greater than 0xff (= 255).

  • You may not use any macros or casting.

Error-checking

Check you code using the included utilities of dlc and btest. These are the same utilities I’ll be using to grade your programs, so make sure you know how to use them!

The binary dlc (which stands for data lab checker) is for checking rules compliance. For example, it will check number of operations, and whether you used any illegal function calls, etc. To use dlc, simply run it and give it the command-line argument of bits.c:

% ./dlc bits.c
Note
 The dlc binary was compiled on a Linux Intel system, and will only run on other Linux systems. (For example, machines in lab 2-170 should work just fine. Avoid Sun Machines running Linux.)

btest.c is for checking whether your functions actually run as they are supposed to. To compile btest, use the Makefile that is included:

% make btest

It will compile your bits.c file in with a testing program. As long as there aren’t any errors in your bits.c file, you can then run the binary btest to see if your code returns the correct values. btest has many command-line options to help you diagnose particular problems, or focus on one function. If you run it without any options, it will simply check all of your functions and report all the erroneous ouput it finds for all of them. When you start the assignment, the list of problems will be large. (Since you won’t have written any solutions yet.) To see what the options are, run btest -h. I will be grading the programs using btest -g.

3 Tips

  • Use the programs that are included! You should know ahead of time what your score is going to be.
